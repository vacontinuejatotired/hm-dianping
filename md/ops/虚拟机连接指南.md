## 连接方式

### 方式一：SSH 命令行（推荐）

```bash
ssh root@192.168.49.130
```

首次连接会提示确认主机指纹，输入 `yes` 后输入密码即可。

### 方式二：配置 SSH 密钥免密登录（推荐）

1. **检查本地是否已有公钥**（Windows  PowerShell）：

```powershell
Get-ChildItem "$env:USERPROFILE\.ssh\*.pub"
```

如果没有输出，则生成密钥：

```powershell
ssh-keygen -t ed25519 -C "your_email@example.com"
```

2. **将公钥复制到虚拟机**（需要输入密码）：

```powershell
type "$env:USERPROFILE\.ssh\id_ed25519.pub" | ssh root@192.168.49.130 "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
```

3. **验证免密登录**：

```bash
ssh root@192.168.49.130 "echo 连接成功"
```

### 方式三：使用 VS Code Remote-SSH

1. 安装 VS Code 插件 `Remote - SSH`
2. 按 `F1` → `Remote-SSH: Connect to Host...` → 输入 `root@192.168.49.130`
3. 输入密码即可连接

## 常见问题

### 1. SSH 连接被拒绝

```bash
ssh: connect to host 192.168.49.130 port 22: Connection refused
```

**原因**：虚拟机未开机或 SSH 服务未启动。

**解决**：在 VMware/VirtualBox 中启动虚拟机，或在虚拟机控制台中执行：

```bash
systemctl start sshd
```

### 2. SSH 解析主机名失败

```bash
ssh: Could not resolve hostname centos: ...
```

**原因**：`~/.ssh/config` 中 `HostName` 配置错误（如写成了 `Centos` 而非 IP 地址）。

**解决**：将 `HostName` 改为正确的 IP 地址 `192.168.49.130`。

### 3. RabbitMQ 连接超时

```bash
java.net.ConnectException: Connection timed out: getsockopt
```

**原因**：虚拟机上的 RabbitMQ 服务未启动或已崩溃。

**解决**：登录虚拟机后执行：

```bash
systemctl start rabbitmq-server
systemctl status rabbitmq-server
```

### 4. RabbitMQ 管理界面无法访问

检查管理插件是否启用：

```bash
rabbitmq-plugins enable rabbitmq_management
systemctl restart rabbitmq-server
```

管理界面地址：`http://192.168.49.130:15672`
默认账号：`guest` / `guest`
