# 商店通用查询接口设计方案

> 状态：待审查  
> 审查通过后方可实现

---

## 1. 背景

现有商店查询接口分散且功能不完整：

| 接口 | 功能 | 不足 |
|------|------|------|
| `GET /shop/of/type` | 按类型分页，支持 Redis GEO 距离排序 | 不支持按价格/评分/名称排序 |
| `GET /shop/near/{typeId}` | 附近热门，固定 5km + 10 条 | 无分页、无排序选择 |
| `GET /shop/of/name` | 按名称模糊搜索 | 无类型过滤、无排序 |

需要一个统一、可扩展的商店查询接口，覆盖按类型分页 + 多字段排序 + 按距离排序。

---

## 2. 接口定义

```
GET /shop/search?typeId={typeId}&current={page}&pageSize={size}&sortBy={field}_{order}&x={lng}&y={lat}&keyword={name}
```

### 2.1 参数说明

| 参数 | 类型 | 必填 | 默认 | 说明 |
|------|------|------|------|------|
| `typeId` | Integer | 是 | - | 商铺类型ID |
| `current` | Integer | 否 | 1 | 页码 |
| `pageSize` | Integer | 否 | 10 | 每页条数 |
| `sortBy` | String | 否 | `default` | 排序字段+方向，见 §2.2 |
| `x` | Double | 按距离时必填 | - | 经度 |
| `y` | Double | 按距离时必填 | - | 纬度 |
| `keyword` | String | 否 | - | 店铺名称模糊搜索 |

### 2.2 sortBy 可选值

| sortBy 值 | 排序方式 | 数据来源 | 备注 |
|-----------|---------|---------|------|
| `default` | 综合排序（按销量降序） | MySQL | 兜底排序 |
| `price_asc` | 均价从低到高 | MySQL | - |
| `price_desc` | 均价从高到低 | MySQL | - |
| `score_asc` | 评分从低到高 | MySQL | - |
| `score_desc` | 评分从高到低 | MySQL | 推荐 |
| `name_asc` | 店名 A→Z | MySQL | - |
| `name_desc` | 店名 Z→A | MySQL | - |
| `distance_asc` | 距离从近到远 | Redis GEO | 需传 x,y |
| `sold_desc` | 销量从高到低 | MySQL | - |

> **扩展方式**：后续新增排序字段只需在 `sortBy` 枚举中加一个值 + 映射一个 `OrderItem`，不改接口签名。

---

## 3. 业务逻辑

### 3.1 处理流程

```
请求 → 参数校验
       │
       ├── sortBy=distance_asc → x,y 必填校验
       │                           │
       │                           ▼
       │                     Redis GEOSEARCH
       │                     shop:geo:{typeId}
       │                     半径 5000m + WITHDIST
       │                           │
       │                           ▼
       │                     按距离升序排列
       │                     MySQL IN(ids) 查完整数据
       │                     手动分页（skip/limit）
       │                     → 填充 distance
       │
       └── 其他 sortBy → MySQL 排序
                           │
                           ▼
                     query().eq("type_id", typeId)
                            .like(keyword, "name", keyword)
                            .orderBy(sortBy → OrderItem 映射)
                            .page(new Page<>(current, pageSize))
                            → 返回 Page<Shop>
```

### 3.2 距离排序说明

- 复用现有 Redis GEO 集合 `shop:geo:{typeId}`
- 搜索半径 5000 米（5km），与 `/shop/of/type` 一致
- GEOSEARCH 返回所有命中结果（不分页），按距离升序
- MySQL `IN(ids)` 按 GEO 返回顺序回查完整数据
- 手动截取 `(current-1)*pageSize` 到 `current*pageSize` 实现分页
- 每条 Shop 填充 `distance` 字段

### 3.3 MySQL 排序说明

- sortBy 参数映射到 MyBatis-Plus `OrderItem` 列表
- 使用 `orderBy` 链式调用，不拼接 SQL，防注入
- 经 keyword 非空时拼接 `like("name", keyword)`

## 4. 返回结构

### 4.1 成功响应

```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "name": "店铺名",
      "typeId": 1,
      "images": "/imgs/shop/1.jpg,/imgs/shop/2.jpg",
      "area": "陆家嘴",
      "address": "上海市浦东新区...",
      "x": 121.5,
      "y": 31.2,
      "avgPrice": 80,
      "score": 45,
      "sold": 5000,
      "comments": 200,
      "openHours": "10:00-22:00",
      "distance": null
    }
  ],
  "total": 50,
  "current": 1,
  "pageSize": 10
}
```

> `distance` 仅 `sortBy=distance_asc` 时不为 null。
> 返回字段当前为 Shop 全部字段，后续可在 Service 层用字段白名单控制。

### 4.2 错误响应

```json
{
  "success": false,
  "errorMsg": "按距离排序时必须传入经纬度(x,y)"
}
```

## 5. 扩展性设计

### 5.1 排序映射表

```java
private static final Map<String, List<OrderItem>> SORT_MAP = Map.ofEntries(
    entry("default",      List.of(orderItem("sold", false))),
    entry("price_asc",    List.of(orderItem("avg_price", true))),
    entry("price_desc",   List.of(orderItem("avg_price", false))),
    entry("score_asc",    List.of(orderItem("score", true))),
    entry("score_desc",   List.of(orderItem("score", false))),
    entry("name_asc",     List.of(orderItem("name", true))),
    entry("name_desc",    List.of(orderItem("name", false))),
    entry("sold_desc",    List.of(orderItem("sold", false)))
);
```

新增排序字段只需加一行映射，`sortBy` 参数透传到接口即可。

### 5.2 后续可能的扩展

- **字段白名单**：新增 `fields` 参数（如 `fields=id,name,score`），Service 层拼接 `select` 链
- **多条件组合过滤**：新增 `minPrice`/`maxPrice`/`minScore` 等参数，拼接 `ge`/`le` 条件
- **分页参数可配置**：`pageSize` 最大值由常量控制

| 现有接口 | 是否保留 | 备注 |
|---------|---------|------|
| `GET /shop/of/type` | 保留 | 兼容存量前端，内部可委托给新接口 |
| `GET /shop/near/{typeId}` | 保留 | 兼容存量前端 |
| `GET /shop/of/name` | 保留 | 兼容存量前端 |
| `GET /shop/{id}` | 保留 | 详情查询，不作改动 |
| `POST /shop` | 保留 | 新增，不作改动 |
| `PUT /shop` | 保留 | 更新，不作改动 |
