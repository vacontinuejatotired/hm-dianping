# 项目优化方案（非 Login 模块）

> 本文档记录架构审查中发现的秒杀模块、商铺模块、Upload、RabbitMQ 等其他模块的问题与优化建议。
>
> 最后更新：2026-06

---

## 秒杀模块

### 🔴 MQ 消费者二次扣减 MySQL 库存与 Lua 扣减 Redis 冲突

- **问题**：`MqVoucherOrderServiceImpl.voucherOrderHandler()` 在 MQ 消费侧又做了一次 MySQL 库存扣减。但 `saveOrder()` 中的 `MqSeckill.lua` 已经在 Redis 侧扣过一次。
- **触发场景**：如果 MySQL 库存数据未与 Redis 同步，MySQL 侧扣减可能失败，导致消息进入重试→死信队列。
- **讨论**：
  - 审查方：MQ 消费者不应再扣减 MySQL 库存，库存追踪应以 Redis Lua 为准。
  - 作者反馈：MySQL 不能只负责插入订单。乐观锁有限制，如果 Redis 挂了还能做一点缓冲。所以 MySQL 侧扣减保留作为兜底。

### 🔴 秒杀下单双流程冲突：两个入口、两套 Lua

- **问题**：`VoucherOrderController` 暴露了两个秒杀入口（V2 `seckill` 和旧版 `saveOrder`），可能造成 Redis 库存被扣两次。
- **当前状态**：开发用测试接口，历史遗留问题，暂不动。

### 🟡 两个 Seckill.lua 并存

- **问题**：`resources/` 下同时存在 `Seckill.lua`（旧版 Redis Stream 方案，未被引用）和 `MqSeckill.lua`（新版 MQ 方案）。
- **建议**：删除 `Seckill.lua`。

## 商铺模块

### 🟠 `ShopController.saveShop()` 和 `updateShop()` 无权限校验

- **问题**：任何人可创建/修改商铺数据。
- **当前状态**：测试代码，不急。

### 🟡 `ShopServiceImpl.queryWithLock()` 同步自旋锁 + 硬编码 sleep

- **问题**：
  - 无自旋上限，热点 key 失效时可耗尽 Tomcat 线程
  - `Thread.sleep(200)` 硬编码 200ms 是调试残留
- **建议**：自旋加次数上限（如 3 次）；移除 `Thread.sleep(200)`。

---

## Upload 模块

### 🟠 `UploadController.deleteBlogImg()` 路径穿越漏洞

- **问题**：仅检查 `isDirectory()`，未检查 `../` 路径穿越，可删除任意文件。
- **建议**：
  ```java
  String canonicalPath = file.getCanonicalPath();
  String uploadDir = new File(SystemConstants.IMAGE_UPLOAD_DIR).getCanonicalPath();
  if (!canonicalPath.startsWith(uploadDir)) {
      return Result.fail("非法的文件名称");
  }
  ```

## RabbitMQ

### 🟡 `RabbitConfig.AlternatEexchange()` 声明了备用交换器但未绑定队列

- **问题**：备用交换器拼写错误（`AlternatEexchange`）且没有队列绑定，不可路由消息被静默丢弃。
- **建议**：要么绑定监控队列并告警，要么去掉备用交换器配置。
