### 2.2 PasswordEncoder 升级

**文件**: `utils/security/PasswordEncoder.java`

替换 MD5+Salt 实现为 BCrypt（纯工具类，无 Spring 依赖）：

```java
import org.springframework.security.crypto.bcrypt.BCryptPasswordEncoder;

public class PasswordEncoder {
    private static final BCryptPasswordEncoder ENCODER = new BCryptPasswordEncoder();

    public static String encode(String rawPassword) {
        return ENCODER.encode(rawPassword);
    }

    public static boolean matches(String rawPassword, String encodedPassword) {
        // 先 bcrypt 匹配
        if (ENCODER.matches(rawPassword, encodedPassword)) {
            return true;
        }
        // 降级 MD5 校验（旧格式，过渡期兼容）
        if (matchesMd5(rawPassword, encodedPassword)) {
            // 匹配成功后由调用方负责升级为 bcrypt
            return true;
        }
        return false;
    }

    private static boolean matchesMd5(String rawPassword, String encodedPassword) {
        if (encodedPassword == null || rawPassword == null) return false;
        if (!encodedPassword.contains("@")) return false;
        String[] arr = encodedPassword.split("@");
        String salt = arr[0];
        String expectedHash = org.springframework.util.DigestUtils.md5DigestAsHex(
                (rawPassword + salt).getBytes(java.nio.charset.StandardCharsets.UTF_8));
        return encodedPassword.equals(salt + "@" + expectedHash);
    }
}
```

**密码自动升级**：密码登录成功后，如果用户密码还是 MD5 格式，会自动用 bcrypt 重新加密并更新数据库：

```java
if (PasswordEncoder.matches(rawPassword, user.getPassword())) {
    // 旧格式密码，升级为 bcrypt
    if (!user.getPassword().startsWith("$2")) {  // 不是 bcrypt → 旧 MD5 格式
        user.setPassword(PasswordEncoder.encode(rawPassword));
        user.setUpdateTime(LocalDateTime.now());
        updateById(user);
    }
    // ... 生成 Token
}
```

### 2.4 密码登录

**文件**: `service/impl/UserServiceImpl.java` — `login()` 方法

当前流程走验证码。密码登录流程：

```
① phone 格式校验
② 密码非空 → 走密码登录
    ├─ 查用户（phone）
    ├─ 用户不存在 → 返回"账号或密码错误"
    ├─ 检查账户是否锁定（Redis key: "login:lock:{phone}"）
    │   └─ 锁定中 → 返回"账户已锁定，请 30 分钟后重试"
    ├─ 检查 IP 频率（Redis key: "login:fail:ip:{clientIP}"）
    │   └─ 超过 5 次/分钟 → 返回"操作太频繁"
    ├─ 校验密码（PasswordEncoder.matches）
    │   ├─ 成功 → DEL 清除失败计数（`login:fail:count:{phone}` + `login:fail:ip:{ip}`） + 生成 Token
    │   └─ 失败 → INCR 失败计数
    │       ├─ 连续失败 >= 10 → SET lock key（30 分钟过期）
    │       └─ 返回"账号或密码错误"
③ 密码为空 → 走验证码登录（现有逻辑不变）
```

**Redis Key 设计**：

| Key | 用途 | TTL |
|-----|------|-----|
| `login:fail:ip:{ip}` | IP 级失败计数 | 60 秒 |
| `login:fail:count:{phone}` | 用户级连续失败计数 | 24 小时（每次失败重置 TTL） |
| `login:lock:{phone}` | 账户锁定标记 | 30 分钟 |

### 2.5 密码修改接口

**文件**: `controller/UserController.java` + `service/impl/UserServiceImpl.java`

**请求 DTO**（新建 `dto/PasswordChangeDTO.java`）：

```java
@Data
public class PasswordChangeDTO {
    @NotBlank(message = "旧密码不能为空")
    private String oldPassword;
    @NotBlank(message = "新密码不能为空")
    private String newPassword;
}
```

> Controller 参数需加 `@Valid` 注解触发校验：`public Result changePassword(@RequestBody @Valid PasswordChangeDTO dto)`

**请求**：

```
POST /user/password
Content-Type: application/json
Authorization: <token>

{
    "oldPassword": "xxx",
    "newPassword": "xxx"
}
```

流程：
```
① Token 鉴权（依赖 RefreshTokenInterceptor 前置鉴权）
② 校验旧密码（bcrypt matches）
③ 新密码强度校验（isPasswordInvalid）
④ bcrypt 加密新密码 → 更新数据库
⑤ 调用 authService.generateTokenPair(userId) 生成全新双 Token + 新 version
   （复用 LoginSetToken.lua 原子覆盖旧值，无需手动 DEL 旧 token）
⑥ 返回新 TokenPair（前端需更新本地存储）
```

### 2.6 密码强度校验

**文件**: `utils/RegexUtils.java`

新增方法：

```java
public static boolean isPasswordInvalid(String password) {
    if (password == null || password.length() < 8) return true;
    // 至少包含一个大写字母、一个小写字母、一个数字
    return !password.matches("^(?=.*[a-z])(?=.*[A-Z])(?=.*\\d).{8,}$");
}
```

## 3. 安全注意事项

| 风险 | 措施 |
|------|------|
| 暴力破解 | IP 限频（5次/分钟）+ 账户锁定（10次/30分钟） |
| 密码泄露 | bcrypt 自动加盐，即使数据库泄露也无法逆向 |
| 旧 Token 作废 | 修改密码后 bump version，所有旧 Token 立即失效 |
| 时序攻击 | BCryptPasswordEncoder.matches() 恒等比较 |
| 错误信息泄露 | 统一返回"账号或密码错误"，不区分用户是否存在 |
