## 第2章 相关技术

### 2.1 双令牌鉴权机制

- Access Token（JWT）：携带 userId + version，jjwt 签发，30min 有效期
- Refresh Token（UUID 随机串，HttpOnly Cookie，7天）
- Token 版本号机制：每次登录/过期刷新生成新版本号，Caffeine 快速拒绝 + Redis 确认
- Lua 脚本保证登录签发和过期刷新的原子性
- 临期刷新（版本号不变）vs 过期刷新（生成新版本号）

### 2.2 多级缓存架构

- **L1 Caffeine 本地缓存**：userinfoCache + tokenValidVersionCache，异步刷新
- **L2 Redis 分布式缓存**：商铺数据逻辑过期方案 + 空值缓存 + 互斥锁
- **L3 MySQL 数据库**：MyBatis-Plus 查询

### 2.3 Lua 脚本在 Redis 中的应用

- 原子性执行特征
- 登录签发：一次脚本写入 accessToken + refreshToken + version 等多条记录
- 过期刷新：脚本内完成校验 + 写入 + 版本号更新
- 减少客户端与 Redis 的网络往返次数

### 2.4 GEO 位置服务技术

- Redis GEO 数据类型（GeoHash + ZSet）
- GEOADD → GEOSEARCH 附近查询
- shop:geo:{typeId} 按类型分 Key
- 配合 MySQL 回查详情 + ORDER BY FIELD 排序

### 2.5 社交互动技术

- **点赞**：Redis ZSet 存储点赞用户 ID + 时间戳
- **Feed 流**：Redis ZSet 为每个用户维护关注者笔记时间线，滚动分页
- **关注关系**：MySQL follow 表维护

### 3.2 双令牌鉴权模块

#### 3.2.1 令牌签发

- jjwt 签发 JWT（HS256，claim: userId + version）
- UUID Refresh Token → Redis → HttpOnly Cookie
- Lua 脚本原子写入

**核心代码**：`AuthServiceImpl.generateTokenPair()`

#### 3.2.2 令牌校验（Caffeine + Redis 两级）

- Caffeine 快速检查 version
- Redis 确认最新版本
- 临期 → 触发 refresh

**核心代码**：`AuthServiceImpl.validateAccessToken()`

#### 3.2.3 令牌自动续期

- 临期刷新：锁内直接写 Redis，版本号不变
- 过期刷新：Lua 脚本原子操作，生成新版本号

**核心代码**：`RefreshTokenInterceptor.preHandle()`

【插图：图3-2 双令牌鉴权流程示意图】

### 3.3 多级缓存模块

#### 3.3.1 商铺缓存读取链路

- Redis 逻辑过期 → 未过期直接返回
- 已过期 + 获取互斥锁 → 异步线程重建
- 已过期 + 未获取锁 → 返回旧数据

#### 3.3.2 缓存穿透/击穿防护

- 空值缓存（短 TTL）防止穿透
- 逻辑过期 + 互斥锁防止击穿
- 更新：先更新 DB → 再删除 Redis 缓存

**核心代码**：`ShopServiceImpl.queryById()` + `CacheClient`

【插图：图3-3 多级缓存读取流程示意图】
